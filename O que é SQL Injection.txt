A injeção de SQL (SQLi) é uma brecha de segurança que ocorre quando um invasor consegue inserir código SQL malicioso em uma entrada de dados de um aplicativo da web. Isso pode resultar na alteração do comportamento das consultas ao banco de dados, permitindo que o invasor visualize, modifique ou exclua dados não autorizados. Essa vulnerabilidade pode expor informações sensíveis dos usuários ou comprometer a integridade dos dados armazenados.

Além disso, em casos mais graves, um ataque de injeção de SQL pode ser usado para comprometer o servidor subjacente ou outras partes da infraestrutura de back-end. Isso pode abrir caminho para ataques de negação de serviço, nos quais o servidor é sobrecarregado com solicitações maliciosas, resultando em uma interrupção do serviço para usuários legítimos.
